# CLAUDE.md — Framework Home

parent: ../CLAUDE.md

> This folder is **infrastructure, not research.** It holds the shared rules every math project depends on. Research content never lives here. Every research project (RWOT, fouriernew, talks, …) is "a project *over* this framework" and points up to these files instead of copying them.

## Start here

- **`CHANGELOG.md`** — what changed recently. If your context might be stale, read this first, then re-read the spec.
- **`USER MANUAL.md`** — the practical how-to (task recipes, session habits, troubleshooting, quick-reference card). Open this when you want to *do* something.
- **`FRAMEWORK — operating manual.md`** — the technical map: what's read/activated when, the skill table, the consolidation ledger. Open this when you want to know *how the system works*.

## The four sources of truth (one per concern)

| Concern | File |
|---|---|
| Format (naming, frontmatter, templates, annotations, statuses) | `math-framework-spec.md` |
| Pedagogy (how Claude explains & writes) | `AI-human mathematical collaboration principles.md` |
| Human discipline (how you use Claude) | `Human principles for AI-assisted mathematical work.md` |
| Orientation (this map) | `FRAMEWORK — operating manual.md` |

## Supporting docs

- `Heuristics and techniques — a working catalog.md` — transferable mathematical "moves"/lenses (a fourth concern: *what to try when stuck*). Grows from `../tao-blog/`.
- `math-framework-human.md` — human-readable intro/essay (non-authoritative; defers to spec + principles).
- `Kodaira guidelines for Claude mathematical notes.md` — source synthesis behind the pedagogy.
- `General strategies.md` — the live friction-log reading practice.
- `codex.md` — Codex's entry point and instruction set (Codex equivalent of CLAUDE.md).
- `agents_md_for_modulized_math_research.md` — pointer doc (philosophy/format now live in the canonical files above).

## Rule

One concern, one file. Nothing here gets restated in a project's CLAUDE.md — projects point here. If you find framework rules copied into a project, replace the copy with a pointer.
